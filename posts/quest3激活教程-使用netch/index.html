前言#

Meta Quest 3 是一款优秀的 VR 一体机，但在国内激活存在网络限制问题。本文将介绍如何使用 Netch 工具，通过 Windows 电脑配置网络代理，实现 Quest 3 在国内的顺利激活。

准备工作#

在开始之前，你需要准备以下内容：

  • Windows 电脑一台
  • 一个路由器
  • VPN 代理

获取代理服务#

首先需要获取一个 VPN 代理服务。可以访问以下任意链接注册，必须支持 UDP 转发，连接 WiFi 时需要对设备进行时间校准，这个校准步骤就同时验证这个 WiFi 是否可以连网，但这个校准是基于 UDP 的，只开启梯子的系统代理只能走 TCP 又不支持 UDP，所以只能上网不能校准时间。绝大部分机场不支持 UDP，你可以咨询机场客服是否支持 UDP 转发，也可以参考这篇文档Quest 网络受限原因 | Meta Quest 中文指南，测试是否支持 UDP。

如果你没有代理，或者代理不支持 UDP，可以考虑使用以下代理，我已经测试过：

  • 龙猫云
  • 老猫云
  • Web3加速器

选最便宜的买就行，买一个月，后续可能会用其他不支持 UDP 便宜的机场，所以不要买贵的。我之前用的机场就不支持 UDP，但是平时使用完全够用，只有在 Quest3 激活是用到过这一次 UDP。

日常使用可以考虑便宜大碗的Glados，目前至少稳定运行了10年了。

机场注册之后注意看一下是否支持导出订阅地址，有一些机场只能使用自己的客户端，那就无法使用 Netch。请一定要注意！！！因为文章可能维护不及时，我也不保证上面的机场是否还支持导出订阅地址。之前使用过万达云的机场，用了一天就因为无法导出订阅地址无法使用了。他家改成只能使用自己的客户端了。

Netch 配置步骤#

下载 Netch#

从Netch 开源主页下载最新版本。

重要提示：使用 Netch 时不能同时开启其他代理软件！如果之前有开启其他代理，请全部关闭后重启电脑。

安装 Netch#

  1. 解压下载的压缩包到任意磁盘根目录
  2. 运行 Netch.exe 文件

安装 Npcap#

前往官网下载 Npcap 并安装Npcap: Windows Packet Capture Library & Driver。

安装时选择：

  • Support raw 802.11 traffic
  • Install Npcap in WinPcap API-Compatible Mode

添加订阅#

  1. 点击上方的"订阅" → “管理订阅”
    1. Responsive Image
  2. 添加订阅链接
    1. Responsive Image

更新服务器#

  1. 点击上方的"订阅" → “更新服务器”
    1. Responsive Image
  2. 如果更新失败，检查订阅链接是否正确，确保关闭所有其他代理软件

选择节点#

  1. 点击"服务器"，选择一个节点，优先选择美国节点
  2. 如果是自己的代理服务，可以多尝试不同节点

选择模式#

点击"模式"，选择 [pcap2socks]Tencent(10.6.0.1)Preset

Responsive Image

配置 DNS#

  1. 点击"设置"
    1. Responsive Image
  2. 切换到"WinTUN"选项卡
  3. 勾选"使用自定义 DNS"
  4. 勾选"代理 DNS"
  5. 将 DNS 改为 1.1.1.1
  6. 点击右下角"保存"

Responsive Image

启动 Netch#

点击右下角的"启动"按钮，等待启动完成。

判断启动成功：

  • “启动"按钮变成"停止”
  • 下方出现绿色圆形标志

软件右侧出现以下终端窗口：

Responsive Image

Quest 3 激活流程#

1. 连接 WiFi#

  1. 启动 Quest 3
  2. 在语言选择界面选择"简体中文"
  3. 选择"在头戴设备中输入 WIFI 信息"
  4. 选择你的路由器 WiFi，不要点击连接，需要后面配置高级选项

2. 配置静态 IP#

  1. 输入 WiFi 密码后，点击"高级选项"

  2. 将 IP 设置从"DHCP"改为"静态"

  3. 配置静态 IP 地址

    1. IP 地址：10.6.0.1
    2. 网关：10.6.0.2
    3. 网络前缀长度：24
    4. DNS1:8.8.8.8

  4. 滚动到页面最下方，点击"连接"

正常情况下点击连接后，Netch 软件旁边的终端窗口会有响应，有一行输出提示某一个 IP 地址设备加入了链接。

3. 系统更新#

连接成功后，Quest 3 会自动进入更新流程。按照我的经验，激活过程中经常断连，所以需要多尝试几次。不要开始更新就把机器扔一边，经常看看。

常见问题处理#

提示"Wrong STUN Server"#

进入设置，将 STUN 服务器改为以下任意一个：

  • stun.voipstunt.com
  • stun.xten.com
  • stun.ekiga.net

没有绿点或显示 NOUDP/UnsupportedServer#

Responsive Image

可能的解决方案：

  1. 刷新状态：点击 NOUDP 刷新几次，如果变成黄点或红点也可能能用
  2. 兼容性问题：进入设置，勾选"TLS AllowInsecure"，保存后重试
    1. Responsive Image
  3. 修改 STUN 服务器：同上面的解决方案
  4. 检查其他软件：确保关闭所有其他代理软件后重启电脑
  5. 检查节点：选择其他节点重试
  6. 重启设备：重启路由器和 Netch
  7. 跳过绿点检查：如果后续 Quest3 能正常更新（进度条会走），可以不用管绿点

提示"找不到 VCRUNTIME140.dll"#

搜索下载"Windows 常用运行库"并安装。
