解决 OpenSSL SSL_connect: Connection was reset in connection to github.com:443

8月 9 2021 Bug 踩坑记录 1 分钟 lesen (Über 197 Wörter)

在向 GitHub 推送博客时，推送失败报了这个错。也不知道是改了什么设置突然报错。SSL 的错之前遇到一次，就是刚开始配置 Git 时用的https协议，每次push都需要重新输入一次密码。改成ssl协议就 OK 了。当时把 Linux 环境的 Git 改了，但是现在的 Windows 下没改，猜测可能和这也有关，于是就把 URL 改了一下，结果还真好了。
在本地仓库的.git文件里找到config文件，打开后将url改为ssl协议，git@github.com:XXX格式的。

将 Hexo 的配置也改了，找到仓库下的_config.yml

添加方法：

  • 在博客根目录的source文件夹下新建README.md文件
  • 在根目录的_config.yml文件中搜索skip_render，并做如下更改

因为在每次hexo g时候，README 文件都会被自动渲染为 HTML 文件，所以在配置文件中告诉渲染器跳过这个文件不要渲染它。

Hexo 和 GitHub 搭建博客以及更换电脑同步博客

7月 20 2021 1 分钟 lesen (Über 174 Wörter)

只要有source文件夹下所有源文件就可以重新部署，按照正常的搭建 Hexo 环境开始搭建，搭建好以后将source文件夹替换即可，需要应用主题就下载主题然后替换。

注意：

  • 主题更换需要更改_config_yml文件
  • _config_yml文件中的部署配置，branch:master就是每次hexo d操作推送的分支。而在命令行每次git push推送的分支是设置的默认分支hexo
