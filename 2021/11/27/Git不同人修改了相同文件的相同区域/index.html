Git 不同人修改了相同文件的相同区域

11月 27 2021 Git 实战 2 分钟 读完 (约 244 字)

不同人修改了文件的相同区域，如果向远端推送，肯定会被拒绝。这时候就需要解决冲突，

首先拉取远端最新的代码，会提示有冲突的文件，

打开冲突的文件，git 会对冲突区域进行标记，<<<<<<到======区域表示远端的代码。======到>>>>>>>表示本地的代码。这时候就需要自己来判断需要哪些代码，也可以增删一些内容，修改完成后将这些标识符号删除，然后保存退出。

git status查看当前状态，提示还有未合并的路径，需要进行commit操作。

及时git push当前代码。
